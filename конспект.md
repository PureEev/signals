# 1. Конспект № 1
---

## 2. Поле
- Поле: **GF(2) = {0, 1}**.  
- Операция сложения — по модулю 2.  
- Вес Хемминга: `ω(x)` — число ненулевых компонент в векторе `x` (в двоичном случае — число единиц).  
- Расстояние Хемминга:
d(x,y) = ω(x + y)
(сложение компонент по модулю 2).

---

## 3. Упрощённая модель цифровой системы связи
Источник данных → (информационные символы) → Кодер → (кодовые символы) → Канал (+ шум) → Декодер → (кодовые символы + ошибки) → Получатель
Пример конкретной передачи:  
`Источник → [00110010111] → Кодер → [0011100011100110110] → Канал → Получатель`

---

## 4. Двоичный симметричный канал
- Переходы:
0 ──(1−p)──► 0
│ │
│ p │
▼ ▼
1 ──(1−p)──► 1
- `p` — вероятность ошибки одного символа (переход 0→1 или 1→0).

---

## 5. Вес и расстояние — примеры и свойства
- Свойство линейного кода: если `x,y ∈ C`, то `z = x + y ∈ C`. Тогда
d(x,y) = ω(x+y) = ω(z) = d(z,0).
- Пример:
x = (0,0,1,2,0,1)
y = (2,0,1,0,0,1)
⇒ d(x,y) = 2

---

## 6. Линейные коды (общее)
- Линейный код `C` над GF(q) — k-мерное подпространство `F_q^n`.  
- Параметры: `(n, k)` — длина кодового слова и размер информационного слова.  
- Размер множества кодовых слов: `q^k` (например, `q=3, k=2 ⇒ 3^2 = 9`).

---

## 7. Порождающая матрица и кодовые слова
- `G` — порождающая матрица размера `k × n`.  
- Информационное слово `m = (m₁, …, m_k)`.  
- Кодовое слово:
c = m · G
(векторное произведение по полю).
- Кодовые слова — линейные комбинации базисных векторов (строк/столбцов матрицы G).

**Пример (n=5, k=2)** — набор кодовых слов:
c1 = 00000
c2 = 10110
c3 = 01011
c4 = 11101
Разложение по базису `e1, e2`:
0·e1 + 0·e2 = c1
1·e1 + 0·e2 = c2
0·e1 + 1·e2 = c3
1·e1 + 1·e2 = c4

---

## 8. Проверочная матрица
- `H` — проверочная матрица размера `(n − k) × n` такая, что
G · H^T = 0.
- Для какого-то вектора `h = (h₁, …, hₙ)` выполняется:
(c, h) = c₁h₁ + c₂h₂ + … + cₙhₙ = 0 (для всех c ∈ C)
- Пример опорного вектора: `h = [0 0 1 1 1]`. В примере выше `n−k = 3`, поэтому H имеет 3 строки.

---

## 9. Минимальное расстояние и исправляющая способность
- Минимальное расстояние кода:
d_min = min { d(x,y) | x ≠ y, x,y ∈ C }.
Для линейного кода эквивалентно `min ω(x)`, `x ≠ 0`.
- Исправляющая способность `t`:
t ≤ floor((d_min − 1) / 2).
— код гарантированно исправит любые ошибки до кратности `t`.

---

## 10. Скорость кода
- Скорость:
R = k / n
- Примеры: `R = 2/5`, `R = 1/3`.

---

## 11. Простой практический пример — повторный (тройной) код
- Правило кодирования:
0 → 000
1 → 111
- Канал — BSC с вероятностью ошибки `p`.
- Стратегия декодирования: «по большинству».
- Ошибка декодирования возникает только если в трёхбитной реплике произойдёт ≥2 ошибок. Вероятность:
P_e = C(3,2) · p^2 · (1−p) + C(3,3) · p^3
= 3·p^2·(1−p) + p^3
- Численный пример при `p = 10^−3`:
- `p^2 = 10^−6`
- `3·p^2·(1−p) = 3·10^−6 · 0.999 = 2.997·10^−6`
- `p^3 = 10^−9`
- `P_e = 2.997·10^−6 + 10^−9 = 2.998·10^−6 ≈ 3.0·10^−6`
- Для сравнения: без кодирования `P_e = p = 10^−3` — гораздо хуже.

## 13. Полезная сводка формул
- Вес: `ω(x)` — число ненулевых компонент.  
- Расстояние: `d(x,y) = ω(x+y)`.  
- Минимальное расстояние: `d_min = min_{x≠y} d(x,y)`.  
- Исправляемость: `t ≤ ⌊(d_min − 1)/2⌋`.  
- Кодовое слово: `c = m · G`.  
- Паритет: `G · H^T = 0`.  
- Скорость: `R = k / n`.
